## Remaining Issues

### P0 — Security & Correctness

1. **Hardcoded API key in `config.yaml` L45**: `api_key_env: "sk-s08jpaornnq79nuw23uxsi18kk1s2w9m65dm464ry1cja1vi"` — the factory's `_api_key()` calls `os.getenv(env_name)`, treating this as an env var name. Since no env var named `sk-...` exists, the remote profile fails on startup. Fix: change to `api_key_env: "OPENAI_API_KEY"` and store the actual key in `.env`.

2. **`.env.example` incomplete**: Root `.env.example` has a commented `GROQ_API_KEY` but the remote profile uses `OPENAI_API_KEY`. Backend `.env.example` has no API key variable. Both need updating.

### P1 — Cleanup & Robustness

3. **Legacy config properties**: `config.py` has `ollama_endpoint` (L127-129) and `embeddings_provider` (L224-226) that reference old `embeddings.*` YAML keys. These are dead code — the active profile system replaced them. Remove.

4. **Factory validation gaps**: `create_embeddings()` doesn't validate `timeout_seconds` being positive like `create_chat_model()` does (it reuses `_common` but `_common` only validates `timeout_seconds > 0` for chat — actually it validates for both, let me re-check). `_common` validates `timeout_seconds > 0` generically. However, neither function validates `temperature` or `max_tokens` ranges. Add bounds validation.

5. **`.gitignore` incomplete**: Only covers `backend/.env` implicitly. Root `.env` is gitignored but `backend/.env.example` should remain tracked. Add `*.env` pattern to cover all `.env` files while keeping `.env.example` tracked.

### P2 — Test Coverage

6. **Expand `test_provider_factory.py`**: Currently only tests `missing model` and `missing remote key`. Need:
   - `create_chat_model()` with `ollama` provider (mock `ChatOllama`)
   - `create_chat_model()` with `openai_compatible` provider (mock `ChatOpenAI`)
   - `create_embeddings()` with `ollama` provider (mock `OllamaEmbeddings`)
   - `create_embeddings()` with `openai_compatible` provider (mock `OpenAIEmbeddings`)
   - `validate_active_profile()` success path
   - Invalid/unsupported provider rejection
   - Timeout validation (negative, zero)

7. **Embedding batch/query test with LangChain fakes**: Verify `aembed_query` and `aembed_documents` are called correctly through the factory-created instances.

8. **RAG endpoint shape tests**: Verify response schemas with mocked providers for both local and remote profiles.

9. **Cache invalidation tests**: Verify profile change invalidates cache keys.
