# Fix: Summary Generation Timeout (120s) Error

## Problem

The `generate_summary()` method in `backend/app/rag/generator.py` times out after 120s when calling Ollama. The root cause is a timeout mismatch:

1. `generate_summary()` calls `self.client.post()` (OllamaClient) which uses httpx with a **60s timeout** (from `embeddings.timeout` config)
2. Tenacity retries 3 times on timeout, adding ~6s of retry wait time
3. Total wall time exceeds 120s, but the `asyncio.wait_for(timeout=120s)` never fires because `httpx.TimeoutException` is NOT an `asyncio.TimeoutError`
4. The exception falls into the generic `except Exception` handler, logging a misleading "timed out after 120s" error

**Note:** `generate_response()` was already fixed — it uses `_post_to_llm()` which bypasses OllamaClient. But `generate_summary()` still uses the old path.

## Solution

Make `generate_summary()` use `_post_to_llm()` (like `generate_response()` already does), and fix exception handling to catch both timeout types.

## Changes

### File 1: `backend/app/rag/generator.py`

**Change 1:** Update `generate_summary()` to use `_post_to_llm()` instead of `self.client.post()`

```python
# Line 170-174: Change from:
res = await asyncio.wait_for(
    self.client.post("/api/generate", payload),
    timeout=self.timeout_seconds
)
data = res.json()

# To:
data = await asyncio.wait_for(
    self._post_to_llm(payload),
    timeout=self.timeout_seconds
)
```

**Change 2:** Fix exception handling to catch `httpx.TimeoutException` alongside `asyncio.TimeoutError`

```python
# Line 179: Change from:
except asyncio.TimeoutError:

# To:
except (asyncio.TimeoutError, httpx.TimeoutError):
```

**Change 3:** Apply same exception fix to `generate_response()` (line 114) and `generate_streaming_response()` (line 235)

```python
# Line 114 and 235: Change from:
except asyncio.TimeoutError:

# To:
except (asyncio.TimeoutError, httpx.TimeoutError):
```

### File 2: No changes needed

- `client.py` — No changes needed; `generate_summary` will no longer use `OllamaClient.post()`
- `config.py` — No changes needed; timeout values are correct
