# Fix: 409 Conflict on Search

## Problem
Search returns 409 Conflict: "The active embedding profile differs from the index; run confirmed reindexing first."

## Root Cause
The ChromaDB collection has an embedding fingerprint that doesn't match the current configuration. This safety check prevents mixing embeddings from different models.

## Current State
- Stored fingerprint in ChromaDB != Current config fingerprint
- This means documents were indexed with different embedding settings

## Solutions

### Option A: Reindex Documents
This will re-embed all documents with the current model:
```bash
# Call the reindex endpoint
curl -X POST http://localhost:8000/documents/reindex
```

### Option B: Check Current Fingerprint
Run this to see the mismatch:
```python
# In Python
from app.utils.config import config
from app.vector_store.chromadb_service import ChromaDBService

vector = ChromaDBService()
print("Current fingerprint:", config.embedding_profile_fingerprint)
print("Stored fingerprint:", vector.collection.metadata.get("embedding_fingerprint"))
print("Index status:", vector.index_status())
```

### Option C: Reset Collection
If you want to start fresh:
```bash
# Delete the chroma_db folder and restart
rm -rf backend/chroma_db
# Then restart and re-upload documents
```

## Files to Check
- `backend/config.yaml` - embedding settings (model, dimension, endpoint)
- `backend/chroma_db/` - stored collection with old fingerprint
