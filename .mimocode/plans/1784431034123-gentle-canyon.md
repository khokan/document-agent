# Debug: Ollama Request Failures

## Problem
Ollama requests are failing with retry errors:
- `[RETRY] Ollama request failed, retrying in 2.0s... (Attempt 1)`
- `[RETRY] Ollama request failed, retrying in 2.0s... (Attempt 2)`
- `[ERR] Failed to generate embedding for text:`

## Root Cause Analysis
The Ollama client (`backend/app/embeddings/client.py`) connects to `http://192.168.105.44:11434` (configured in `config.yaml` line 23). The retry logic uses tenacity with 3 attempts and exponential backoff.

Possible causes:
1. **Ollama server not running** at the configured endpoint
2. **Network unreachable** - the IP `192.168.105.44` may not be accessible
3. **Ollama model not pulled** - `nomic-embed-text` model may not be downloaded
4. **Timeout too short** - 60s timeout may not be enough for large embeddings

## Status
- Ollama is running on remote server at `192.168.105.44:11434` (confirmed by user)
- Models (nomic-embed-text, mistral) are available on the server (confirmed)
- **CONFIRMED: Server unreachable from this machine** (curl timed out)

## Root Cause
The Ollama server at `192.168.105.44:11434` is not accessible from the machine running the document agent. This causes all embedding requests to fail and retry until timeout.

## Solution: Fix Network Connectivity

### Step 1: Verify Ollama server is running
On the server (192.168.105.44), check:
```bash
# Check if Ollama process is running
ps aux | grep ollama

# Check if port 11434 is listening
netstat -tlnp | grep 11434
# or
ss -tlnp | grep 11434
```

### Step 2: Check firewall on server
```bash
# Check UFW status (Ubuntu)
sudo ufw status

# Check firewalld (CentOS/RHEL)
sudo firewall-cmd --list-all

# Temporarily allow port 11434 (Ubuntu)
sudo ufw allow 11434/tcp

# Or disable firewall temporarily for testing
sudo ufw disable
```

### Step 3: Verify network connectivity
```bash
# From this machine, test basic connectivity
ping 192.168.105.44

# Test if port 11434 is reachable
Test-NetConnection -ComputerName 192.168.105.44 -Port 11434
```

### Step 4: Check if both machines are on same network
```bash
# On this machine
ipconfig

# On server
ip addr show
```
Verify both have IPs in the same subnet (e.g., 192.168.105.x).

### Step 5: Test Ollama API directly
Once connectivity is fixed:
```bash
curl http://192.168.105.44:11434/
# Should return: "Ollama is running"

curl -X POST http://192.168.105.44:11434/api/embeddings \
  -H "Content-Type: application/json" \
  -d '{"model": "nomic-embed-text", "prompt": "test"}'
```
