# Fix: Summary Generation Timeout — Final

## Root Cause

`generate_summary()` still calls `self.client.post()` (OllamaClient, 60s httpx timeout + 3x retries). When httpx times out, `httpx.TimeoutException` is NOT an `asyncio.TimeoutError`, so it falls into the generic `except Exception` handler — logging "timed out after 120s" misleadingly. Meanwhile `generate_response()` already uses `_post_to_llm()` but also has the wrong exception type.

```python
async def _post_to_llm(self, payload):
    url = f"{self.client.base_url.rstrip('/')}/api/generate"
    async with httpx.AsyncClient(timeout=httpx.Timeout(float(self.timeout_seconds))) as client:
        response = await client.post(url, json=payload)
        response.raise_for_status()
        return response.json()
```
