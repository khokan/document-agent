# Frontend vs Backend Alignment Audit

## Executive Summary

The frontend is **significantly behind** the backend. It only uses ~50% of available backend endpoints and is missing major features: chat, RAG query with answers, summarization, and streaming. There are also broken API calls to non-existent endpoints and field name mismatches.

## Critical Issues Found

### 1. Frontend Calls Non-Existent Endpoints

**File:** `frontend/document-agent-ui/src/config/constants.js`

```javascript
SEARCH_ADVANCED: '/search/advanced',   // ‚ùå Does NOT exist in backend
SEARCH_STATS: '/search/stats',         // ‚ùå Does NOT exist in backend
```

**File:** `frontend/document-agent-ui/src/services/api/health.js` (mislabeled ‚Äî contains searchAPI)

```javascript
async generateAnswer(query) {
  return await client.post(`${ENDPOINTS.SEARCH}/answer`, { query });
  // ‚ùå POST /search/answer does NOT exist in backend
}
```

These will cause 404/405 errors if called.

### 2. Dashboard Stats Field Name Mismatch

**File:** `frontend/document-agent-ui/src/pages/Dashboard.jsx`

Frontend expects:
- `stats.total_pages` ‚Äî ‚ùå Backend returns `total_chunks` (not pages)
- `stats.processing_documents` ‚Äî ‚ùå Backend doesn't return this field

Backend actually returns (`GET /documents/stats`):
```json
{
  "total_documents": 5,
  "total_chunks": 142,
  "total_size_mb": 12.34,
  "embedding_dimension": 768,
  "collection_name": "company_documents"
}
```

### 3. Documents Page Field Name Mismatch

**File:** `frontend/document-agent-ui/src/pages/Documents.jsx`

Frontend expects:
- `doc.pages` ‚Äî ‚ùå Backend returns `page_count`
- `doc.size` ‚Äî ‚ùå Backend doesn't return file size in document list
- `doc.id` ‚Äî ‚öÝÔ∏è Backend uses `document_id` (fallback `doc.document_id` exists)

Backend actually returns (`GET /documents`):
```json
{
  "documents": [{
    "document_id": "report_2024",
    "filename": "Report2024.pdf",
    "upload_date": "2024-07-09T10:30:00",
    "status": "indexed",
    "chunk_count": 42,
    "page_count": 15
  }],
  "total_count": 1,
  "total_chunks": 42
}
```

### 4. API Service Files Are Mislabeled

| File | Contains | Should Contain |
|------|----------|----------------|
| `services/api/health.js` | `searchAPI` | `healthAPI` |
| `services/api/index.js` | `healthAPI` | barrel export or `searchAPI` |

The imports work because of how they're used, but the naming is confusing.

### 5. Missing RAG Endpoints in Frontend Config

**File:** `frontend/document-agent-ui/src/config/constants.js`

No endpoints defined for:
- `/rag/query`
- `/rag/chat`

- `/rag/stream`

## Recommended Fixes

### Priority 1: Fix Broken Endpoints (Critical)

**File:** `frontend/document-agent-ui/src/config/constants.js`
- Remove `SEARCH_ADVANCED` and `SEARCH_STATS` (don't exist in backend)
- Add RAG endpoints:
  ```javascript
  RAG_QUERY: '/rag/query',
  RAG_CHAT: '/rag/chat',
  RAG_SUMMARIZE: '/rag/summarize',
  RAG_STREAM: '/rag/stream',
  DOCUMENTS_REINDEX: (id) => `/documents/reindex/${id}`,
  DOCUMENTS_CLEANUP: '/documents/cleanup',
  DOCUMENTS_HEALTH: '/documents/health',
  ```

### Priority 2: Fix Field Name Mismatches

**File:** `frontend/document-agent-ui/src/pages/Dashboard.jsx`
- Change `stats.total_pages` ‚Üí `stats.total_chunks`
- Change `stats.processing_documents` ‚Üí remove or use `stats.total_chunks`

**File:** `frontend/document-agent-ui/src/pages/Documents.jsx`
- Change `doc.pages` ‚Üí `doc.page_count`
- Change `doc.size` ‚Üí remove (not in backend response)
- Ensure `doc.document_id` is used consistently

### Priority 3: Fix Mislabeled Files

- Rename `services/api/health.js` ‚Üí `services/api/search.js` (contains searchAPI)
- Rename `services/api/index.js` ‚Üí `services/api/health.js` (contains healthAPI)
- Or restructure the barrel exports

### Priority 4: Remove Dead Code

**File:** `frontend/document-agent-ui/src/services/api/health.js`
- Remove `generateAnswer()` method (calls non-existent `POST /search/answer`)
- Remove `advancedSearch()` method (calls non-existent `POST /search/advanced`)

### Priority 5: Add Missing RAG Features (New Pages)

This is the biggest gap. The backend supports full RAG but the frontend has no UI for it.

**New files needed:**
1. `pages/Chat.jsx` ‚Äî Multi-turn chat page using `POST /rag/chat`
2. `pages/Query.jsx` ‚Äî RAG query page using `POST /rag/query` (question ‚Üí answer + sources)
