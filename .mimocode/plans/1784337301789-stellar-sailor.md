## Issues Found

### Critical: Hardcoded API Key in config.yaml
**File**: `backend/config.yaml:45`
```yaml
api_key_env: "sk-s08jpaornnq79nuw23uxsi18kk1s2w9m65dm464ry1cja1vi"
```
This is an **actual API key**, not an environment variable name. Must be changed to reference an env var like `OPENAI_API_KEY`.

### Missing: Comprehensive Validation
The current `validate_active_profile()` only checks if models can be instantiated. It doesn't validate:
- `api_key_env` is actually an env var name (not a key value)
- `base_url` is a valid URL format
- `dimension` is a reasonable embedding dimension (e.g., 128-4096)
- `timeout_seconds` is positive
- `temperature` is in valid range (0-2)
- `max_tokens` is positive

### Missing: Profile Info API Endpoint
No endpoint exists to:
- Query current active profile settings (without exposing secrets)
- List available profiles
- Validate a profile configuration

### Step 2: Add Comprehensive Validation to factory.py
**File**: `backend/app/ai/factory.py`

Add validation functions for each field:

```python
def _validate_url(url: str, kind: str, field: str) -> None:
    """Validate URL format."""
    from urllib.parse import urlparse
    parsed = urlparse(url)
    if not parsed.scheme or not parsed.netloc:
        raise ProviderConfigurationError(
            f"Active {kind} provider '{field}' is not a valid URL: {url}"
        )

def _validate_temperature(temp: float, kind: str) -> None:
    """Validate temperature is in valid range."""
    if not 0.0 <= temp <= 2.0:
        raise ProviderConfigurationError(
            f"Active {kind} provider temperature must be between 0.0 and 2.0, got {temp}"
        )

def _validate_max_tokens(tokens: int, kind: str) -> None:
    """Validate max_tokens is positive."""
    if tokens <= 0:
        raise ProviderConfigurationError(
            f"Active {kind} provider max_tokens must be positive, got {tokens}"
        )

def _validate_dimension(dim: int, kind: str) -> None:
    """Validate embedding dimension is reasonable."""
    if not 128 <= dim <= 4096:
        raise ProviderConfigurationError(
            f"Active {kind} provider dimension must be between 128 and 4096, got {dim}"
        )
```

Update `create_chat_model()` and `create_embeddings()` to call these validators.

### Step 3: Add Profile Validation Endpoint
**File**: `backend/app/api/routes.py`

Add new endpoints:

```python
@router.get("/ai/profile", tags=["AI"])
async def get_active_profile():
    """Get current active AI profile info (secrets redacted)."""
    from app.utils import config
    return {
        "active_profile": config.active_ai_profile,
        "chat": {
            "provider": config.chat_settings.get("provider"),
            "model": config.chat_settings.get("model"),
            "base_url": config.chat_settings.get("base_url"),
        },
        "embeddings": {
            "provider": config.embedding_settings.get("provider"),
            "model": config.embedding_settings.get("model"),
            "dimension": config.embedding_settings.get("dimension"),
        },
    }

@router.get("/ai/profiles", tags=["AI"])
async def list_profiles():
    """List all available AI profiles."""
    from app.utils import config
    profiles = config.get("ai.profiles", {})
    return {
        "available": list(profiles.keys()),
        "active": config.active_ai_profile,
    }

@router.post("/ai/profile/validate", tags=["AI"])
async def validate_profile():
    """Validate the active profile can instantiate models."""
    from app.ai.factory import validate_active_profile
    try:
        validate_active_profile()
        return {"status": "valid", "profile": config.active_ai_profile}
    except Exception as e:
        return JSONResponse(
            status_code=422,
            content={"status": "invalid", "error": str(e)}
        )
```
